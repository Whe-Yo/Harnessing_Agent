**안티패턴**: 설치 절차에 복사할 파일을 개별 나열하면, 실행 구성(settings.json)에 파일이 추가될 때 나열 목록 갱신을 잊는다. 두 목록이 어긋나면 해당 기능층이 **침묵 사망**한다 — 에러가 없어서 아무도 모른다.

**실증(260702)**: setup/SKILL.md가 훅 5종만 나열, settings.json은 7종 참조 → 문서대로 신규 설치하면 카운터·자문층(track-tools·process-status) 전체 무효. 검증 게이트도 나열된 것만 테스트해서 못 잡음.

**처방**: ① 나열 대신 글롭(`*.sh` 전부) ② 설치 후 "실행 구성이 참조하는 파일이 모두 존재하는가" 대조 ③ 검증 게이트는 층별 기능 테스트(가드뿐 아니라 자문층도).
